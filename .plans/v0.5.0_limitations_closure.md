# APMODE v0.5.0 — Known-Limitations Closure Plan (consolidated)

Status: consolidated, 2026-04-17
Scope: close all 10 items from README "Known Limitations" for v0.5.0 final
Inputs: architectural review + literature cross-check (Gibiansky 2008, Bräm 2023, Janssen 2024, Mager & Jusko 2001, Peletier & Gabrielsson 2012, Daxberger 2021)
Supersedes: the draft dated 2026-04-17 (pre-consolidation)

## 1. Scope decision — FINAL

**v0.5.0 includes M0 + M1 + M1.5 + M2a + M2b + M3 + M4** with M3 scope-reduced to low-dim latent/input-layer RE only (block ≤16×16). No RE on full NN weights in v0.5.0. This keeps every Known Limitation closable (truthfully), not just documented-away.

**Hard defers (ADR-gated, loud-reject + CLI + bundle manifest + README/CHANGELOG disclosure):**

- **#2 NODE infusions (RATE>0)** — `docs/adr/0004-node-infusions.md`. Explicit documentation line: *"NODE backend is oral-only for v0.5.0."*
- **Stan steady-state (SS!=0) algebra, general or narrow** — `docs/adr/0003-stan-ss-scope.md`. No narrow-scope SS work this release. Stan/NODE hard-reject SS at Gate 1.

**VI for NODE RE (Janssen 2024 CPT:PSP, "Mixed effect estimation in deep compartment models")** — v0.5.1 ADR trigger, not a v0.5.0 pivot. VI adds a new estimator class with new tuning surface and credibility implications; scope-blown for 7–8 weeks.

## 3. M0 — ScoringContract + bundle schema bump (prerequisite)

**New Pydantic model in `src/apmode/bundle/models.py`:**

```python
class ScoringContract(BaseModel):
    model_config = ConfigDict(frozen=True)
    contract_version: Literal[1] = 1
    nlpd_kind: Literal["conditional", "marginal"]
    re_treatment: Literal["integrated", "conditional_ebe", "pooled"]
    nlpd_integrator: Literal[
        "nlmixr2_focei",
        "laplace_blockdiag",
        "laplace_diag",
        "hmc_nuts",
        "none",
    ]
    blq_method: Literal["none", "m1", "m3", "m4", "m6_plus", "m7_plus"]
    observation_model: Literal["additive", "proportional", "combined"]
    float_precision: Literal["float32", "float64"]
```

**DiagnosticBundle**: add `scoring_contract: ScoringContract`. Required, not optional. Every backend runner populates before emitting the bundle.

**Backend defaults:**
| Backend | nlpd_kind | re_treatment | nlpd_integrator | float_precision |
|---|---|---|---|---|
| nlmixr2 | marginal | integrated | nlmixr2_focei | float64 |
| bayesian_stan | marginal | integrated | hmc_nuts | float64 |
| jax_node (pooled) | conditional | pooled | none | float32 |
| jax_node (Laplace diag) | conditional | conditional_ebe | laplace_diag | float32 |
| jax_node (Laplace block-diag) | conditional | conditional_ebe | laplace_blockdiag | float32 |

**Gate-3 enforcement (hard-separate leaderboards):** survivors grouped by **exact ScoringContract equality**. No cross-contract composite ranking. Removes the draft contradiction where M4 claimed "mixed survivor set" produces a 3-component composite — that path is explicitly deleted. Gate 3 emits one ranking JSON per contract class; the reporter surfaces each as a separate leaderboard.

**Submission-lane dominance rule** (closes the "three 1-candidate leaderboards → no recommended" governance gap):

```
In Submission lane ONLY:
  recommended_candidate_id is eligible iff
    scoring_contract.re_treatment == "integrated"
    AND scoring_contract.nlpd_kind == "marginal"
  Else: recommended_candidate_id = None, report emits explicit warning.
```

In Discovery and Optimization lanes, separate leaderboards are the final output (no "recommended" dominance rule).

**Versioning:**
- `_COMPLETE_SCHEMA_VERSION` in `src/apmode/bundle/emitter.py` → 2.
- `policies/*.json` schema_version → 0.5.0 for gate; 2.2.0 for profiler.
- `CLAUDE.md` AUTO markers resync via `scripts/sync_readme.py`.

**Tests:**
- `tests/unit/test_scoring_contract.py` — frozen behavior, equality, JSON roundtrip, Literal validation
- `tests/integration/test_gate3_contract_enforcement.py` — mixed contracts → separate leaderboards (never mixed composite)
- `tests/integration/test_submission_dominance_rule.py` — three contracts in submission → only integrated+marginal eligible for `recommended`

## 7. M2b — NODE scaling + RNG/layout lock (weeks 2–3, MUST complete before M3)

| # | Item | Effort | Files | Deliverable |
|---|---|---|---|---|
| 4 | Per-subject scaling | M | `backends/node_trainer.py:140`, `backends/node_runner.py` | Replace Python for-loop with **`jax.lax.map` over subjects** (per-subject ODE solves remain ragged-safe). `jax.vmap` fastpath ONLY if all subjects share a uniform time grid — never over ragged dose schedules. Non-`lax.map` fallback retained behind a flag for debugging. |

**Engineering invariants:**
- **Pad/mask only in the loss computation**, never in the ODE RHS or adjoint path. Diffrax adjoint accumulates gradient on phantom timepoints if they enter the integrator.
- **RNG derivation:** `subject_keys = jax.vmap(lambda i: jax.random.fold_in(master_key, i))(jnp.arange(n_subjects))`. Never reuse a subject-level key across within-subject forward passes. `apply_random_effects()` audited for key-reuse bugs.
- **Equivalence test tolerance:** per-subject **relative** ratio on AUC/Cmax, threshold 1e-3, not absolute 1e-6 (float32 ODE-accumulation reality).
- **Memory ceiling documented:** A100-40G holds ~2000 subjects for a 4-compartment NODE. Any `MAX_VMAPPED` constant is a latency/debug knob, not a memory ceiling; default `None`.

**Tests:**
- `tests/unit/test_node_vmap_equivalence.py` — `lax.map` vs for-loop, per-subject rel ratio ≤ 1e-3
- `tests/integration/test_node_perf_200.py` — 200-subject run under regression budget
- `tests/unit/test_node_rng_isolation.py` — subject key determinism; fold_in audit

Exit: pooled NLL matches pre-change baseline; >50-subject runs no longer truncated.

## 8. M3 — NODE random effects (weeks 4–6, high risk, scope-reduced)

| # | Item | Effort | Files | Deliverable |
|---|---|---|---|---|
| 3 | Per-subject RE | L | `backends/node_trainer.py`, new `backends/node_random_effects.py` | Laplace approximation with **block-diagonal Hessian as primary**, diagonal Hessian as secondary fallback. RE only on **low-dim latent / input-layer parameters** (block ≤16×16). No RE on full-NN weights in v0.5.0. |

**Optimization + fallback hierarchy (bounded):**

1. **Primary:** L-BFGS with line search to find RE mode; block-diagonal Hessian at mode. (Daxberger 2021 "Laplace Redux" — L-BFGS captures off-diagonal curvature through gradient history.)
2. **Fallback 1 (single):** Fixed ridge penalty added to Hessian blocks + `sigma_re` clamp ≥ 1e-6. No trust-region zoo.
3. **Fallback 2:** Mark `re_status` accordingly (see enum below) and **exclude from cross-paradigm ranking**. Pooled-NLL diagnostics ARE still written to the bundle with explicit `ScoringContract.re_treatment="pooled"` so the candidate appears on the pooled-only leaderboard — never silently dropped from the bundle.

**`re_status` enum (distinguishes failure modes; replaces overloaded "unsupported"):**
- `not_applicable` — model has no RE by design
- `supported_not_run` — policy/lane chose pooled; RE codepath intentionally skipped
- `fit_ok` — Laplace converged
- `failed_non_convergence` — optimizer did not converge
- `failed_non_pd_hessian` — Hessian not PD even after ridge
- `failed_numerical` — NaN/Inf during Hessian assembly
- `unsupported_architecture` — RE requested on disallowed parameterization

**Divergence-expectation checklist (for divergence tests):**
- Near-zero true omega: clamp `sigma_re ≥ 1e-6` pre-Hessian.
- Stiff ODE + high ETA: retry with Kvaerno5 (stiff) before declaring divergence.
- Boundary-hitting structural parameters: policy-configurable prior widths.

**Tests:**
- `tests/unit/test_node_laplace.py` — pooled golden (true omega=0 → RE≈0 + PD Hessian)
- `tests/integration/test_node_re_recovery.py` — synthetic RE recovery on 100-subject fixture
- `tests/unit/test_node_re_fallback.py` — each failure mode produces distinct `re_status`, excluded from cross-contract ranking, bundle carries pooled entry

## 15. Resolved open questions

| Question | Decision | Rationale |
|---|---|---|
| Scoring-contract granularity | Seven Literals (see §3). `nlpd_integrator` distinguishes FOCEI vs Laplace-diag vs Laplace-blockdiag vs HMC-NUTS vs none. | Architectural — captures every axis of NLPD comparability at bundle-emission time. |
| Contract-class grouping | Hard-separate leaderboards. No cross-contract composites. | Conditional vs marginal vs integrated likelihoods are on incomparable scales (PRD §10 Q2). |
| TMDD QSS parameterization | Pure-MM (3 params) is default; parallel-linear+MM only on ambiguous evidence. Gated by profiler linear-component detection. | Gibiansky 2008 identifiability analysis — parallel-linear form is overparameterized for pure TMDD. |
| LSS threshold | Policy knob (`policies/*.json/profiler_thresholds/tmdd_lss_ratio_threshold`, default 1.25). Not hardcoded. | Threshold is not a literature-backed value; must be tunable per lane. |
| VI vs Laplace for NODE RE | Laplace-only in v0.5.0; Janssen 2024 triggers a v0.5.1 ADR. | VI adds a new tuning surface and credibility implications out of scope for a 7-8 week window. |
| `re_status` semantics | Seven-valued enum (see §8); no overloaded "unsupported". | Distinct failure modes must be distinguishable for audit and ranking-exclusion logic. |
| Submission-lane dominance | `re_treatment=="integrated" AND nlpd_kind=="marginal"` eligible for `recommended`. | Only integrated-marginal contracts satisfy the regulatory evidence standard PRD §4.3.1 requires. |
